# Database Architecture

Yohan.AI menggunakan **Supabase PostgreSQL** sebagai database utama.

Seluruh modul aplikasi berbagi satu sumber data yang konsisten dengan pemisahan menggunakan schema PostgreSQL.

Database dirancang untuk:

- Konsisten
- Mudah dikembangkan
- Mendukung AI
- Aman dengan Row Level Security (RLS)
- Siap untuk multi-tenant pada fase berikutnya

# Design Principles

## Single Source of Truth

Supabase adalah sumber data utama untuk seluruh platform.

Tidak ada data bisnis yang disimpan secara permanen di frontend.

---

## Modular Schema

Database dipisahkan berdasarkan domain bisnis.

Contoh:

- customer
- property
- sales
- chat
- ai
- automation

Pendekatan ini membuat struktur lebih mudah dipelihara dan dikembangkan.

---

## Security First

Setiap tabel harus mempertimbangkan:

- Authentication
- Authorization
- Row Level Security (RLS)
- Auditability

---

## AI Ready

Struktur database dirancang agar dapat mendukung:

- Lead Scoring
- Buyer Behavior Intelligence
- Recommendation Engine
- Predictive Analytics

tanpa perlu perubahan besar pada fondasi database.

# Database Responsibilities

Database bertanggung jawab untuk:

- Menyimpan data bisnis
- Menjaga integritas data
- Menyediakan query yang efisien
- Mendukung audit trail
- Menjadi fondasi AI Intelligence

---

# Migration Strategy

Seluruh perubahan struktur database harus dilakukan melalui migration.

Workflow:

```text
Design

↓

Migration

↓

Review

↓

Apply

↓

Verification

↓

Documentation
```

Migration langsung pada Production Database tanpa file migration tidak diperbolehkan.
