# Development Rules

Semua task wajib mengikuti urutan berikut:

1. Planning
2. Review Architecture
3. Coding
4. Build
5. Runtime QA
6. Documentation
7. Merge

Tidak boleh langsung merge tanpa Build PASS.

---

# Build Checklist

Sebelum commit:

- TypeScript PASS
- ESLint PASS
- Production Build PASS

Perintah:

```bash
npm run build
```

Jika gagal:

Merge dibatalkan.

# Documentation Rule

Mintlify adalah dokumentasi resmi.

Dokumen lokal hanya digunakan sementara bila diperlukan.

Semua perubahan besar harus diperbarui di Mintlify.
