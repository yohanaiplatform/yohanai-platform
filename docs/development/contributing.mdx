# Contributing Guide

Terima kasih telah berkontribusi pada Yohan.AI.

Dokumen ini menjelaskan standar pengembangan yang harus diikuti oleh seluruh developer maupun AI Assistant.

---

# Development Philosophy

Yohan.AI dikembangkan dengan prinsip:

- Simplicity
- Maintainability
- Scalability
- Consistency
- Documentation First

---

# Before You Start

Sebelum mengerjakan task:

- Baca Start Here
- Baca Project Context
- Baca Project Status
- Pastikan memahami sprint aktif

---

# Standard Workflow

```text
Task

↓

Architecture Review

↓

Implementation

↓

Code Review

↓

Build Verification

↓

Runtime QA

↓

Git Commit

↓

Push

↓

Documentation Update
```

---

# Coding Standards

Seluruh kode harus mengikuti:

- TypeScript Strict Mode
- ESLint
- Prettier
- Project Coding Standards

---

# Pull Request Checklist

Sebelum merge:

- Build berhasil
- TypeScript tanpa error
- Tidak ada console error
- Runtime QA selesai
- Dokumentasi diperbarui

---

# Commit Message Convention

Gunakan Conventional Commits.

Contoh:

```text
feat: add CRM lead detail page
fix: resolve dashboard loading issue
docs: update AI module documentation
refactor: simplify property service
chore: update dependencies
```

---

# Documentation Rule

Jika mengubah:

- Architecture
- Database
- API
- Module
- Workflow

maka dokumentasi Mintlify wajib diperbarui.

---

# Code Review Principles

Review mencakup:

- Readability
- Performance
- Security
- Scalability
- Consistency

---

# Quality Gate

Kode hanya boleh di-merge apabila:

- Build PASS
- Runtime PASS
- Documentation Updated

---

# Golden Rule

Selalu tinggalkan project dalam kondisi lebih baik daripada saat mulai mengerjakannya.
