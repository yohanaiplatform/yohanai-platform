# Architecture Decision Records (ADR)

Dokumen ini mencatat seluruh keputusan arsitektur penting yang diambil selama pengembangan Yohan.AI.

Tujuan ADR adalah memberikan konteks mengapa suatu keputusan dibuat, alternatif yang dipertimbangkan, serta dampaknya terhadap sistem.

# Decision List

## ADR-001 — Mintlify as Single Source of Truth

**Status**

Accepted

### Decision

Seluruh dokumentasi resmi proyek disimpan di Mintlify.

### Reason

- Mudah diakses seluruh tim.
- Mendukung version control.
- Mengurangi dokumen lokal yang tidak sinkron.

### Impact

- Dokumentasi Markdown lokal tidak lagi menjadi referensi utama.
- Semua pembaruan dokumentasi dilakukan melalui repository dokumentasi.

---

## ADR-002 — Supabase as Primary Backend

**Status**

Accepted

### Decision

Supabase digunakan sebagai backend utama.

### Reason

- PostgreSQL
- Authentication
- Storage
- Row Level Security
- Realtime
- Edge Functions (future)

### Impact

Seluruh modul menggunakan schema Supabase sebagai sumber data utama.

---

## ADR-003 — Next.js as Frontend Framework

**Status**

Accepted

### Decision

Frontend dibangun menggunakan Next.js dengan TypeScript.

### Reason

- Modern React framework
- Server Components
- Routing
- Performance
- Developer Experience

---

## ADR-004 — Modular Architecture

**Status**

Accepted

### Decision

Platform dipisahkan menjadi modul independen.

### Modules

- Dashboard
- CRM
- Property
- Sales
- AI Intelligence
- Communication Automation

### Impact

Setiap modul dapat dikembangkan tanpa memengaruhi modul lain secara langsung.

---

## ADR-005 — AI as Decision Support

**Status**

Accepted

### Decision

AI berfungsi sebagai pendukung keputusan, bukan pengambil keputusan.

### Principles

- AI memberikan insight.
- AI memberikan rekomendasi.
- Agen tetap mengambil keputusan akhir.

---

## ADR-006 — Documentation First

**Status**

Accepted

### Decision

Perubahan arsitektur signifikan harus diikuti dengan pembaruan dokumentasi.

### Impact

Dokumentasi selalu mencerminkan kondisi sistem yang sebenarnya.
